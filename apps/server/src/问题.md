# 已知问题与规划（跟踪清单）

## ✅ 已完成（无需再按旧方案改造）

- **多 Agent 与任务步**：计划 `steps[].assignedAgentId`、当前 `running` 步、`resolveEffectiveAgent`、Web 端「锁定 Agent」(`agentLocked`) 已落地；详见 `agentRegistry.ts`、`chatProcessing.ts`、`ChatPage.tsx`。
- **内置工具扩展方式**：`builtinProvider` 通过 `getToolSchemas()` / `getTool(name)` 动态对接，**新增工具在 `agent/tools/index.ts` 注册即可**，不必在 `builtinProvider.ts` 逐条写死名称。
- **MCP 工具注册**：请求路径在 `chatProcessing.ts` 中通过 `getMcpProvidersForRegistry()` 并入 `createRegistryWithProviders`；配置见 MCP 相关模块。`mcpClient.ts` 中 stub 仅为占位说明，主流程使用真实 SDK 客户端。
- **Trace 文件生命周期**：按日 `trace-YYYY-MM-DD.jsonl`；超过 `ONECLAW_TRACE_FILE_MAX_BYTES`（默认 64MB）时轮转为 `trace-YYYY-MM-DD-partN.jsonl`；写入后节流清理早于「今天 − `ONECLAW_TRACE_RETENTION_DAYS`（默认 30）」天的文件。`traceQuery.listTraceFiles` 已支持同日多段文件扫描。
- **CLI trace 筛选**：`trace failed` 支持 `--since`、`--tool`、**`--code`（errorCode）**、`--days`；`trace replay` 等逻辑不变。
- **滚动摘要**：`buildModelContext` 在超 token 预算时按 **`ONECLAW_CHAT_ROLLING_MERGE_CHUNK`（默认 15）** 批量折入摘要；`foldDroppedIntoRolling` 同步使用该阈值。
- **滚动预跑（进阶）**：助手回复写入后，若 **`ONECLAW_ROLLING_PREFETCH_ENABLED`（默认 true）**，后台调用与 `buildMessagesForModel` 相同逻辑更新 `rollingSummary` / `archivedMessageCount`，下一轮请求前 **`awaitRollingPrefetchIdle`**，减少同步摘要等待。见 `session/rollingPrefetch.ts`。
- **Skills**：`skills/` 支持 `.json` 与 `.md`（`---` frontmatter）；`.md` 中 **`tools` 支持多行 JSON**（`extractMultiLineField`）；解析失败带路径与原因。
- **权限外置**：`ONECLAW_DATA_DIR/policy-overrides.json` 合并覆盖各 profile；见 `policyOverrides.ts`。
- **listTasks**：`tasks-index.json` 索引 + 按需读全文；`rebuildTaskIndexFromDisk()`。
- **计划步骤失败策略**：`PlanStep.onStepFail`（`fail_task` / `ask_user` / `goto_step`）；`daily_report` 模板示例。
