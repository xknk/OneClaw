# OneClaw TUI 新手教程

本教程面向**第一次使用终端界面（TUI）**的用户，说明如何启动 OneClaw、如何与界面交互、如何使用斜杠命令，以及和「命令行 / 仓库根目录」的关系。

---

## 1. TUI 是什么？

**TUI**（Text User Interface）是在**终端**里运行的图形化界面：用字符、边框和颜色展示「聊天窗口、输入框、状态栏」，而不是浏览器网页。

- **优点**：不打开浏览器，在本机终端里直接对话；与项目里的 **REPL**、**WebChat** 使用同一套对话处理逻辑（`handleUnifiedChat`）。
- **要求**：终端窗口**尽量大一些**（建议高度 ≥ 25 行、宽度 ≥ 80 列），否则布局会挤、需要依赖终端滚动查看历史。

---

## 2. 启动前需要准备什么？

1. **已进入本仓库根目录**（包含 `package.json` 的那一层）。
2. 已安装依赖：`pnpm install`（若尚未安装）。
3. 可选：已配置 `.env`（模型、数据目录等），首次使用可运行：
   ```bash
   pnpm cli onboard
   ```
   完成初始化与目录说明。

---

## 3. 怎样进入 TUI？

在**仓库根目录**打开终端，任选一种方式：

| 方式 | 说明 |
|------|------|
| `pnpm cli` | **无子命令时默认进入 TUI**（与 `pnpm cli tui` 等价） |
| `pnpm cli tui` | 显式进入 TUI |
| `pnpm oneclaw` | 通过 `bin/oneclaw.cjs` 调用，等价入口 |
| `oneclaw` | 若已全局 `pnpm link` 且 PATH 正确 |

常用参数：

```bash
pnpm cli tui
pnpm cli tui -p 18789 --session mychat
pnpm cli tui --agent <AgentID> --task <taskId>
```

- **`-p` / `--port`**：本机 WebSocket 端口（默认 `18789` 或环境变量 `ONECLAW_TUI_WS_PORT`）。
- **`--session`**：会话键（默认 `cli`），用于区分不同对话上下文；**不要**和正在跑的 `pnpm dev` Web 端共用同一个 session，以免混写历史。
- **`--agent` / `--task`**：可选，与 Agent、任务工作流关联时使用。

---

## 4. 界面分区说明（从上到下）

1. **顶栏**：项目名称、版本、本机 WebSocket 地址（如 `ws://127.0.0.1:18789`）、当前 `session` 提示。
2. **聊天区**：用户消息以 `>` 开头，助手以 `●` 开头，系统提示以 `!` 开头。
3. **斜杠命令菜单**（输入 `/` 后出现）：在**输入框上方**弹出，可键盘选择。
4. **输入区**：分隔线、提示文案、带边框的输入行；**反色块**一般为光标。
5. **底栏**：连接状态、模型名、`streaming`/`idle`、快捷键提示等。

对话过程中可能出现 **`● generating response…`**，表示正在等待模型回复。

---

## 5. 普通对话（最基础）

1. 用鼠标或键盘**焦点**落在输入框（多数终端点击即可）。
2. **输入文字**，按 **Enter** 发送。
3. **不要**在句首误加 `/`，否则会被当成命令；普通聊天直接输入中文或英文即可。

---

## 6. 斜杠命令 `/`（内置命令）

在输入框**行首**输入 **`/`（斜杠）**，会弹出命令列表（菜单），**无需**先背命令。

### 6.1 打开菜单与筛选

- 输入 **`/`**：列出全部内置命令。
- 输入 **`/h`**：会筛选出以 `/h` 开头的项（如 `/help`）。
- **行首是 `/` 且中间没有空格**时才会显示菜单；若已输入 `/session xxx`，则不再弹出菜单，按整行命令处理。

### 6.2 菜单里的键盘操作

| 按键 | 作用 |
|------|------|
| **↑ / ↓** | 在列表中上下移动 |
| **Tab** | 把当前选中项**填入输入框**（补全命令名） |
| **Enter** | 若当前输入**未**完整匹配某条命令，则先**补全**；若已完整匹配且**不需要参数**，则执行该命令 |
| **Esc** | 清空当前输入 |

### 6.3 内置命令一览（与当前代码一致）

| 命令 | 说明 |
|------|------|
| `/help` 或 `/?` | 显示帮助说明 |
| `/session <key>` | 切换会话键（若启动时带了 `--task`，转录仍可能按任务隔离，见帮助文案） |
| `/clear` | 清空**本窗口**聊天记录（不删服务端持久化文件） |
| `/status` | 当前会话、模型、目录等摘要 |
| `/onboard` | 运行初始化（等价于在仓库根执行 `pnpm cli onboard`） |
| `/doctor` | 运行自检（等价于 `pnpm cli doctor`） |
| `/task …` 或 `/t …` | 执行任务子命令，例如 `/task list`、`/t get <id>` |
| `/trace …` 或 `/tr …` | 执行 trace 子命令，例如 `/trace dir` |
| `/start` | 仅提示：应在**独立终端**运行 `pnpm cli start` 启动 Gateway |
| `/exit` 或 `/quit` | 退出 TUI |

**需要参数的命令**（如 `/session`）：若只输入 `/session` 就回车，会提示用法，需要写成 `/session 我的会话名` 或先 `/session` 再跟空格和参数（视当前实现提示为准）。

---

## 7. 在 TUI 里「执行命令行」是什么意思？

TUI 里**一部分**斜杠命令会在**后台**调用仓库里的 CLI（通过 `pnpm cli …`），**输出以系统消息形式显示在聊天区**，而不是新开一个黑窗口。

例如：

- `/doctor` → 等价于在仓库根执行 `pnpm cli doctor`
- `/task list` → 等价于 `pnpm cli task list`
- `/trace failed -S 24h` → 等价于带同样参数的 `pnpm cli trace …`

**注意**：

- 请在**仓库根目录**启动 TUI（与 `pnpm cli` 脚本一致），否则子进程可能找不到项目。
- Windows 下会使用 `pnpm.cmd` 执行；若失败，界面会显示错误摘要。

---

## 8. TUI 与 REPL、Web、命令行的关系

| 入口 | 说明 |
|------|------|
| **TUI** | 本机 WebSocket + Ink 界面；默认 `pnpm cli` 无参数即进入 |
| **REPL** | `pnpm cli repl`，纯文本对话，无 Ink 界面 |
| **Gateway** | `pnpm cli start` 或 `pnpm run dev`，浏览器 WebChat 等 |
| **仅 CLI** | `pnpm cli task …`、`pnpm cli trace …` 等，不进入 TUI |

同一套对话逻辑在 **TUI / REPL / Web** 中应一致；**sessionKey** 与 **task** 的用法请以 `/help` 与 `/status` 为准。

## 9. 常见问题（简要）

1. **连不上 / 一直 connecting**  
   检查端口是否被占用；可换 `pnpm cli tui -p <其他端口>`。

2. **界面闪烁**  
   后台组件（如 MCP）若向终端直接打日志，可能与 Ink 冲突；当前版本在 TUI 模式下会尽量抑制这类输出。若仍异常，可开大终端窗口或缩小同时运行的任务。

3. **退出**  
   输入 **`/exit`**，或 **`Ctrl+C`**（按界面提示）。

4. **历史记录太长**  
   聊天区会随内容变长，**由终端缓冲区滚动**查看更早记录；也可用 **`/clear`** 只清本窗口显示。

---

## 10. 建议的新手流程

1. 仓库根：`pnpm install` 后执行 `pnpm cli onboard`（或 TUI 内 `/onboard`）。  
2. `pnpm cli doctor` 或 TUI 内 `/doctor`，确认模型与目录正常。  
3. `pnpm cli` 进入 TUI，先随便发一句「你好」测试。  
4. 输入 **`/`** 熟悉菜单与 **↑↓ Tab Enter**。  
5. 需要任务 / 排错时，再使用 `/task …`、`/trace …`。
