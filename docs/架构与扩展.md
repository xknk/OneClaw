# 架构与扩展

模块边界与扩展方式（原 `architecture/plugin-boundaries.md` 的实用摘要）。

## 1. 模块边界

| 模块 | 职责 | 边界要点 |
|------|------|----------|
| **Gateway** | HTTP、路由、鉴权、会话、调 Agent、写回 | 只认 `UnifiedInboundMessage` / `UnifiedOutboundMessage`，不解析各渠道原始协议细节 |
| **ChannelAdapter** | 入站解析、出站发送 | `parseInbound` / `sendOutbound`；当前有 WebChat、QQ（OneBot v11） |
| **Agent Runtime** | 多轮推理与工具循环 | 只依赖消息列表与 `ToolSchema[]` / `Tool[]` |
| **Tools** | 内置能力 + MCP + Runtime Skill | 经 `ToolRegistry` / `ToolExecutionService` 执行 |
| **Skills** | 工作区 JSON 技能包 | 提供额外 ToolSchema 与可选 `systemPrompt` |

配置统一走 `appConfig`（`src/config/evn.ts`），业务避免直接读零散 `process.env`。

## 2. 新增渠道

1. 实现 `ChannelAdapter`：`channelId`、`parseInbound`、`sendOutbound`。
2. 在网关创建/路由处注册，解析成功后走统一会话 + Agent，再用同一 Adapter 回写。
3. 密钥与 token 单独环境变量，并写明安全边界（见 [安全配置与风险](./security-risks.md)）。

## 3. 新增 / 使用 Skill

1. 在 `workspaceDir/skills`（默认见 [使用说明](./user-guide.md)）下放置 JSON。
2. 格式：单对象或 `{ "skills": [ ... ] }`；字段含 `id`、`tools`、`systemPrompt` 等。
3. **条件启用**：在 Skill 上配置 `enableWhen`（如 `channelIds`、`sessionKeyPrefixes`、`keywordsAny`、`agentIds`）。聊天入口使用 `loadSkillsForContext` / `getCombinedToolSchemasForContext`（见 `src/skills/loadSkills.ts`、`src/agent/withSkills.ts`）。未配置 `enableWhen` 的技能视为全局启用。
4. 工具执行：与内置工具同名可覆盖描述；新工具名需在运行时有对应 **Tool** 实现，否则会报未知工具。
