# 终端交互（本机 REPL）实施方案

> **状态**：**阶段 A、B 已落地**（`src/cli/repl.ts`、`pnpm cli repl`）。阶段 C（流式、TUI）仍为可选。使用说明见 [使用说明 · 终端对话](./user-guide.md#终端对话repl)。

## 1. 目标

在**不通过** `POST /api/chat`、不强制用户手写 `curl/fetch` 的前提下，提供与 OpenClaw / 本地 Cloud Code 类产品相近的体验：

- 在终端中持续输入多轮对话；
- 与 WebChat **共用**同一套聊天处理链（工具、Skills、会话、任务上下文、审批闸门等）；
- **可选**：与 `pnpm dev` 网关**同时**运行（需注意会话文件争用，见 §5）。

## 2. 设计原则

| 原则 | 说明 |
|------|------|
| **复用核心逻辑** | 每轮调用 `handleUnifiedChat(inbound, sendOutbound)`（定义见 `src/server/chatProcessing.ts`），与 `src/server/createServer.ts` 中 `/api/chat` 路径一致。 |
| **仅替换 I/O** | HTTP 的 `req.body` / `res.json` 替换为：**readline 读 stdin** + **stdout 打印回复**；不复制 Agent 循环或工具执行代码。 |
| **独立进程** | REPL 为单独 CLI 进程（`tsx src/cli.ts repl` 或 `pnpm repl`），不嵌入 Express；便于本机调试与不启动网关时使用。 |
| **配置同源** | 入口 `import "dotenv/config"`（`src/cli.ts`），复用 `appConfig`、模型与 workspace 配置。 |

## 3. 用户侧形态（已实现）

### 3.1 命令入口

- 主命令：`pnpm cli repl` 或 `pnpm repl`（`package.json` 脚本）。
- 可选参数：
  - `--session <key>`：初始会话键，默认 `cli`。
  - `--agent <id>`：`UnifiedInboundMessage.agentId`。
  - `--task <taskId>`：`taskId`，与 WebChat 行为对齐。
  - `-v` / `--verbose`：stderr 打印 `traceId` 与完整 `metadata`。

### 3.2 交互循环

- 提示符：`> `。
- 内置命令：
  - `/help`、`/?`：帮助；
  - `/session <key>`：运行时切换 `sessionKey`（带 `--task` 时转录仍以 `task:<id>` 为主）；
  - `/clear`：清屏（不删对话历史）；
  - `/status`：打印当前 `sessionKey` / `taskId` / 数据目录与 Ollama 模型摘要；
  - 单独输入 `/` 回车：列出可用命令；**行首 `/` + Tab**：readline 补全；
  - `/exit`、`/quit`：退出；
  - `Ctrl+C`：退出（SIGINT）。

### 3.3 输出

- 默认：stdout 打印模型回复文本。
- `-v`：stderr 打印 `[traceId] …` 与 `[meta] {…}`（含 `traceId`、`agentId`、`profileId`、`taskId`），便于 `pnpm cli trace get <traceId>`。

## 4. 技术实现要点

### 4.1 入站消息构造

| 字段 | 当前值 | 说明 |
|------|--------|------|
| `channelId` | `"webchat"` | 与 Profile / 策略绑定一致。 |
| `channelUserId` | `"cli-local"` | 固定 CLI 用户标识。 |
| `sessionKey` | 启动参数或 `/session` 切换 | 与转录存储一致。 |
| `text` | 用户输入行 | |
| `agentId` / `taskId` | CLI 可选 | 与 HTTP body 对齐。 |

### 4.2 出站回调

将 `outbound.text` 写入 stdout；`-v` 时把 `metadata` 写入 stderr。

### 4.3 代码位置

- `src/cli/repl.ts` → `runRepl()`
- `src/cli.ts` → `program.command("repl")`

### 4.4 依赖

仅 Node 内置 `readline/promises`，无额外 npm 依赖。

## 5. 与网关并行时的风险

- **会话 / 转录文件**：相同 `ONECLAW_DATA_DIR` 且相同 **有效** 会话键时并发写入可能交错。建议 Web 用 `main`，REPL 默认 `cli`。
- **MCP**：两进程各自子进程与连接，属正常。
